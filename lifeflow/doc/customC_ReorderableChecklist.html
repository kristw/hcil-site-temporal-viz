Custom Components: Reorderable Checklist

Concept

Users can drag and drop to change the order of items in the list. The component will trigger events when checked, order changed or data (such as text in the figure belof) changed. The number of checkboxes is also customizable. This component is implemented using swing’s JTable with custom CellRenderer, CellEditor and TransferHandler (for drag and drop).

Figure 1: LegendTable: An example of reorderable checklist component

Implementation

The code is in package edu.umd.cs.hcil.lifeflow.components.legend. There are seven classes:

  • Legend: Component to draw in each row
  • LegendModel: Data model for the Legend component (extends from custom class CheckableModel)
  • LegendModelTransferable: Wrap LegendModel for drag and drop transfer
  • LegendTable: The checklist (extends from swing’s JTable)
  • LegendTableChangeListener: Declare listener when checked, order changed, or model changed.
  • LegendTableModel: Contains list of LegendModel. It extends from custom class CheckableTableModel, which extends from swing’s DefaultTableModel.
  • LegendTransferHandler: Describes drag and drop action

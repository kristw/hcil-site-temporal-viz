Chapter 1: Overview

1.1 Design

DataKitchen is designed for the conversion of the original data set into the format LifeFlow uses. Check out the page for the details of LifeFlow format.

The original data file can be too large to be loaded into memory and apply changes efficiently. Therefore, DataKitchen loads only a small fraction (e.g. the first 1000 rows) of the input data to create a "preview" of the data and the modification being made. The users can apply changes and get a glance at the result in the preview. The actual changes, which applies to all the rows in the input file, are being made when the users save the file.

1.2 Login

After executing DataKitchen, a dialog will prompt the users to enter their name in the textfield. The user name will be logged in the log file, which keeps the modifications being made to the original data. Type in the user name in the textfield and click the button "OK" will bring up the main window of DataKitchen.

1.3 Preview Panel

In the Preview Panel shows the preview created after loading the input file. The preview is shown as a table with column names on top and each row representing an instance of original input. Columns are colered in yellow when blank. The lable on the tab shows the file name and in the bottom status bar shows the number of rows in the original input data file.

1.4 Instruction Panel

Each change to the original data is considered to be an "instruction". The users can see the all the modifications being made to the data in the Instruction Panel as a list of instructions.

1.5 Provenance Panel

Since the data may come from various data source, information about the data itself is kept for the users to check later on. In the Provenance Panel, the owner, source, date, and last modifying date can be edited by the users. The users are also suggested to put down their comments on the data. Though the meta-data about the data is not required to enable the conversion of data, it can be helpful to organize many data files the users may have.

The file extension used by the provenance file is .prov. DataKitchen will load automatically < data file name >.prov, or create the file if not exist. It is suggested that the provenance file being passed along with the data file when the users share the data file with other users.

To change the values in the fields, simple type the information and click the "Save" button.

1.6 Open and save data files

The button to open/save the data file can be found in the File menu on the menu bar. Following "File->Open File..." and "File->Apply and Save As..." will bring up a dialog respectively to open and save data file.

1.6.1 Open data file

  1. Select the file or type in the file name in the textfield. When a file is selected, the contents of the first ten rows will show in the bottom of the dialog.
  2. Select the separater between columns from the pull-down menu. Default is tab.
  3. Check/Un-check "First row is column name". When checked, the value in the first row will be treated as the name of the columns.
  4. Select proper time format from the pull-down menu, or click the radiobutton and type the format in the textfield.
  5. Click the button "Open".
Note that the settings are crucial to the conversion of data file.

Figure 1.3: The procedure of openning a data file.

1.6.2 Save data file

DataKitchen will save the modified data file into the format LifeFlow supports. Note that the modification to the data will NOT be made until the data file is saved.
  1. Select the file or type in the file name in the textfield.
  2. Click the button "Save".

Appendix A: Upgrade Java™ Runtime Environment

  • Windows

    • To check the active version of the Java™ Runtime Environment for Windows, type java -version in the Windows command line, and press Enter

  • Mac
    1. Launch Terminal.app, type java -version, and press the return key. Notice that this computer does not have the required version of JRE active:
      2. new-host:~ karenlee$ java -version
java version "1.4.2_22"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.4.2_22-b02-329)
Java HotSpot(TM) Client VM (build 1.4.2-92, mixed mode)
    2. Locate, and install, the most current version of Java for Mac OS X 10.x, available from http://support.apple.com/, where 10.x is your version of Mac OS X. [See Note below*]
    3. Launch Java Preferences.app, and confirm that the most recent version of Java is checked On, and that it appears first in the list:

      4. Figure A.1: The Mac Java Preferences.

    4. In Terminal.app, again run the java -version command.
    5. Confirm that the minimum required version of Java, or higher, is now active:
      6. new-host:~ karenlee$  java -version
java version "1.6.0_26"
Java(TM) SE Runtime Environment (build 1.6.0_26-b03-384-9M3425)
Java HotSpot(TM) 64-Bit Server VM (build 20.1-b02-384, mixed mode)

* This step is current as of August 2011. However, the method for updating JRE for Mac OS X may change in the future. Please check the apple.com for the latest instructions. [The October, 2010 Java Development Guide for Mac OS X states: Note: As of the release of Java for Mac OS X 10.6 Update 3, the Java runtime ported by Apple and that ships with Mac OS X is deprecated. Developers should not rely on the Apple-supplied Java runtime being present in future versions of Mac OS X. The Java runtime shipping in Mac OS X 10.6 Snow Leopard, and Mac OS X 10.5 Leopard, will continue to be supported and maintained through the standard support cycles of those products.]
